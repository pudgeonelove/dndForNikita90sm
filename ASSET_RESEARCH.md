# Asset Research

The current build keeps all class, enemy, NPC, and item visuals self-contained with inline SVG so the GitHub Pages copy does not depend on hotlinked images.

Checked options:

- DiceBear HTTP API: consistent generated SVG avatars, useful if remote avatar URLs are acceptable.
- OpenGameArt CC0 portraits: useful public-domain game-art pool, but each asset still needs individual style and license review.
- Pixabay fantasy portraits: large image pool under the Pixabay Content License, but photo and illustration styles vary heavily.

Recommended direction for this prototype: keep the inline SVG visual system for consistency, then replace specific portraits later only with locally downloaded and credited assets.
